# Writing Guidelines

When writing documentation:

- Never use em dashes (—). Use commas, periods, or parentheses instead.
- Write in plain English.
- Prefer short sentences.
- Avoid corporate jargon and marketing language.
- Explain concepts as if writing for an experienced developer who is new to the project.
- Use descriptive headings.
- Prefer examples over long explanations.
- Keep paragraphs under 5 lines.
- Use bullet lists where they improve readability.
- Prioritize clarity over sounding sophisticated.
